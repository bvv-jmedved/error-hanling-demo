# Retrospective Prompt

**Purpose:**
This file defines the standard retrospective prompt to be used after each completed and committed feature.
It ensures that lessons learned are captured consistently and can be fed back into conventions, documentation, or design decisions.

---

## Prompt

> Give me a short retrospective summary:
> – What slowed it down?
> – What conventions or assumptions were unclear?
> – What could be automated, clarified, or standardized for next time?

Output to retrospective in folder .codex/retrospective with a reference to the Use Case or Technical Debt issue solved as UC-xxx-yyyyy-retrospecttive.md or TD-xxx-yyyyyy-retrospective.md respectively.
